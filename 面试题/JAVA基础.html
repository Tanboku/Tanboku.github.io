# JAVA基础面试题

# Java 中的序列化和反序列化是什么？

序列化就是把内存中的Java对象转换成二进制字节流，这样才能存到硬盘或者通过网络发给其他的机器。反序列化就是反过来，把字节流还原成Java对象。硬盘和网络只认0和1，不认Java对象，所以必须有这么一层转化。

主要就是三点：怎么做、怎么防、怎么稳

怎么做：类必须实现 Serializable 接口，这就是个许可证，没有它JDK不会让你序列化。

怎么防：敏感字段比如密码，加个 transient 关键字，序列化的时候自动跳过。

怎么稳：显示定义 serialVersionUID，相当于给类打个版本戳，防止改了类结构后反序列化旧数据时报错。

public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    
    private String username;
    private transient String password; // 不参与序列化
    private int age;
}

# Java 中 Exception 和 Error 有什么区别？

Exception 和 Error 都是 Throwable 的直接子类，只有继承了 Throwable 的对象才能 throw 和 catch。

核心区别在于：

  1. Exception 代表程序运行过程中可以预料、可以恢复的异常情况，属于业务逻辑范畴，应该也能够被合理处理，比如 NullPointerException、IOException 等
  2. Error 代表 JVM 本身或者系统环境的严重错误，程序通常无法恢复，出现后往往意味着进程需要终止或重启，比如 OutOfMemoryError、StackOverflowError 等

简单说，Exception 是可以被处理的程序异常，Error 是系统级的不可恢复错误。

Exception 又分为 Checked Exception 和 Unchecked Exception：

  1. Checked Exception：继承自 Exception 但不继承 RuntimeException，编译时必须显式处理，要么 try-catch 要么 throws 声明抛出，比如 IOException、SQLException
  2. Unchecked Exception：继承自 RuntimeException，不需要显式捕获，运行时才抛出，比如 NullPointerException、IndexOutOfBoundsException

# 什么是 Java 的多态特性？

父类引用指向子类对象，调用同一个方法时，会执行子类自己的实现。

Animal dog = new Dog();     // 父类引用指向子类对象
Animal cat = new Cat();
dog.shout();   // 输出：汪汪   → 运行时才知道调用 Dog 的 shout()
cat.shout();   // 输出：喵喵   → 运行时才知道调用 Cat 的 shout()

# 你认为 Java 的优势是什么？

跨平台：跨平台是 Java 最经典的卖点，「Write once, Run anywhere」。代码编译成字节码后，JVM 负责翻译成不同操作系统的机器指令。同一套代码在 Windows 上开发，直接扔到 Linux 服务器跑，不用改一行。

自动内存管理：Java 自带垃圾回收机制（GC）。不像 C++ 那样需要程序员手动申请和释放内存（容易忘释放导致内存泄漏，或者误释放导致崩溃）。Java 会自动在后台帮你打扫内存垃圾，程序员只需要专注于业务逻辑，开发效率高多了。

生态强大：Java 的生态圈太庞大了。不管你想做什么功能，基本上都能找到现成的、成熟的开源库（比如 Spring 全家桶）。这就像搭积木，别人把积木都造好了，我拿来拼就行，不用重复造轮子。

工程化友好：工程化友好指的是 Java 的强类型、面向对象封装、继承、多态这些特性，代码结构清晰，适合几百人协作的大型项目。

# Java 中的参数传递是按值还是按引用？

在 Java 中，参数传递只有按值传递，不论是基本类型还是引用类型。

1）基本数据类型如 int、char、boolean 等：传递的是值的副本，也就是基本类型的数值本身。对方法参数的任何修改都不会影响原始变量。

2）引用数据类型如对象引用：传递的是引用的副本，也就是对象引用的内存地址。方法内可以通过引用修改对象的属性，但不能改变引用本身，使其指向另一个对象。

# 说说 Java 中 HashMap 的原理？

HashMap 底层就是一个数组，结合链表和红黑树来解决冲突。

它的工作原理其实很简单，主要说这 3 点：

1）怎么存

存一个 Key-Value 时，先算 Key 的 hashCode，然后用（table.length-1）& hash 算出应该放在数组的哪个下标位置。

2）冲突了怎么办

两个不同的 Key 算出来下标一样，就叫哈希冲突。HashMap 的解决办法是用链表把它们串起来，JDK 8 做了优化，如果同一个下标下的链表超过 8 个节点（且数组长度大于等于 64），就转成红黑树，查找速度从 O(n) 变成 O(logn)。

3）扩容机制

数组长度是有限的，存多了会挤。HashMap 有个负载因子默认 0.75，数组用了 75% 就自动扩容，把数组大小翻倍，然后把所有数据重新计算放到新数组里。这个操作叫 rehash，比较耗性能，所以初始化时最好给个预估容量。
