# RabbitMQ面试题

# RabbitMQ 怎么实现延迟队列？

RabbitMQ 原生并不支持延迟消息，但可以通过两个特性组合实现：TTL（消息存活时间）加上 DLX（死信交换器）。另外也可以用官方提供的延迟消息插件来实现。

TTL + DLX 方案的核心思路是这样的：给正常队列设置消息存活时间，但不配置消费者。消息在队列里待到过期后，会自动转发到绑定的死信交换器，消费者从死信队列里取消息，延迟效果就出来了。

生产者 → 发送 → 交换机 → 路由 → 正常队列 → 超时 → 死信队列（实际是死信交换机）→ 消费者

如果插件的话更直接，安装 rabbitmq-delayed-message-exchange 插件后，创建一个延迟类型的交换器，发消息时带上延迟时间参数，RabbitMQ 会自动在到期后才投递到队列。

# RabbitMQ 中消息什么时候会进入死信交换机？

消息进入死信交换机主要有三种情况：

1）消费者主动拒绝消息。调用 basic.reject 或 basic.nack 时把 queue 设为 false，消息就不会重新入队，直接丢给死信交换机。

2）消息过期了还没被消费。RabbitMQ 支持给消息或队列设置 TTL，比如设置 30 秒、30 秒内没人消费，这条消息就变成死信被转走。

3）队列满了装不下。队列设置了 x-max-length 或 x-max-length-bytes 限制后，新消息进来时如果超出容量，队头最老的消息会被挤到死信交换机。
